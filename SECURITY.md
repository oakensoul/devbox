# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via
[GitHub Security Advisories](https://github.com/oakensoul/devbox/security/advisories/new).

Include as much of the following information as possible:

- Type of issue (e.g., privilege escalation, credential exposure, command injection)
- Full paths of source file(s) related to the issue
- Step-by-step instructions to reproduce the issue
- Impact of the issue and how an attacker might exploit it

## Response Timeline

- **Acknowledgement:** within 72 hours of report
- **Triage and initial assessment:** within 1 week
- **Fix or mitigation:** as soon as possible, depending on severity

## Scope

This policy covers the `oakensoul/devbox` repository, including:

- The CLI and library code in `src/`
- GitHub Actions workflows in `.github/`
- Configuration and preset handling

## Disclosure

We follow coordinated disclosure. We will work with you to understand and
address the issue before any public disclosure.
